# 推荐插件

| 插件或集成             | 推荐等级           | 建议作用域     | 为什么装                                                                                                                                                          |
| ---------------------- | ------------------ | -------------- | ----------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| Codex Security         | 必装               | 用户级或项目级 | 官方专门用于扫描授权代码，并进一步确认可能成立的漏洞发现。适用于不同语言和项目，属于通用安全补充。([OpenAI 开发者][1])                                            |
| GitHub                 | 强烈建议           | 用户级         | Codex 本身已经能操作本地 Git；GitHub 集成的价值是处理 Git 本身覆盖不到的 PR、Issue 等远程对象。你采用 PR 流程时很有用。([OpenAI 开发者][2])                       |
| Browser                | Web 项目建议启用   | Codex 内置     | Codex App 已捆绑 Browser 插件，可操作本地开发服务器和文件页面，用于复现 UI 问题及验证修复。一般前端验证先用它，不必立即加 Playwright。([OpenAI 开发者][3])        |
| Figma                  | 前端/UI 项目建议   | 项目级         | 可以让 Codex 访问设计稿，为页面和组件实现提供真实设计上下文。纯后端、CLI、基础设施项目无需安装。([OpenAI 开发者][2])                                              |
| Context7               | 强烈建议           | 用户级         | 为 Codex提供更新的第三方库和框架文档，适合技术栈经常变化的开发者。官方将其列为推荐 MCP；若插件目录中有 Context7，就安装插件版，否则配置 MCP。([OpenAI 开发者][2]) |
| Playwright             | 自动化测试按需     | 项目级         | MCP,当内置 Browser 不足以完成浏览器自动化、流程重现或端到端检查时再配置。不要因为是 Web 项目就默认安装。([OpenAI 开发者][2])                                      |
| Chrome Developer Tools | 复杂浏览器调试按需 | 项目级         | MCP,可控制并检查 Chrome。通常与 Playwright 二选一开始使用，避免向 Codex 暴露多套功能重叠的浏览器工具。([OpenAI 开发者][2])                                        |
