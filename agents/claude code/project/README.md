# Claude Code 项目级配置使用指南

本目录存放 Claude Code 项目级 `CLAUDE.md` 母版。

项目级配置只记录当前项目特有、且 Claude 无法从代码、构建文件、工具配置和现有文档中可靠推断的信息。

## 推荐初始化顺序

新项目建议按照以下顺序初始化：

1. 创建 Git 仓库。
2. 使用项目脚手架生成基础代码。
3. 安装基础依赖并确认项目能够正常构建或启动。
4. 将本目录中的 `CLAUDE.md` 复制到目标项目根目录。
5. 填写已经明确的项目约束，删除不适用的占位章节。
6. 在目标项目中启动 Claude Code。
7. 执行 `/init`，让 Claude 根据当前代码库检查项目配置。
8. 人工审查并整理最终的 `CLAUDE.md`。

推荐顺序可以概括为：

```text
创建项目脚手架
    ↓
确认项目可以运行
    ↓
复制 CLAUDE.md 模板
    ↓
填写已知项目约束
    ↓
执行 /init
    ↓
人工审核和精简
```

不建议在空仓库中直接执行 `/init`，因为此时缺少构建文件、依赖配置、目录结构和现有代码，Claude 能识别的信息有限。

## 复制模板

在仓库根目录执行：

```bash
cp "agents/claude code/project/CLAUDE.md" <project>/CLAUDE.md
```

也可以手动复制到目标项目根目录：

```text
<project>/CLAUDE.md
```

## 执行 `/init`

复制并初步填写模板后，在目标项目中启动 Claude Code：

```bash
claude
```

然后执行：

```text
/init
```

完成后，应人工检查生成或建议补充的内容，不要未经审核直接接受。

可以继续向 Claude 提出以下要求：

```text
请基于当前代码库检查并完善 CLAUDE.md。

要求：

1. 保留现有模板的职责边界。
2. 只补充无法从代码和配置中可靠推断的重要信息。
3. 不重复用户级 CLAUDE.md 中已经存在的通用规则。
4. 不复述完整目录树、依赖列表或构建配置。
5. 不确定的内容不要自行编造。
6. 删除未填写或不适用的占位章节。
7. 保持文件简洁、明确、可执行。
```

## 已有项目

对于已经具备完整代码、构建配置和测试配置的项目，可以直接：

1. 复制项目级 `CLAUDE.md` 模板。
2. 填写已知的项目约束。
3. 启动 Claude Code。
4. 执行 `/init`。
5. 审查并精简最终内容。

不要为了填写模板而重复抄写项目中已经存在的信息。

## 项目级配置应该包含什么

适合写入项目级 `CLAUDE.md` 的内容包括：

- 项目的标准验证命令及其适用范围。
- 无法由工具自动检查的代码约定。
- 模块之间的依赖方向。
- 不允许手工修改的生成代码。
- 必须保持兼容的接口或数据结构。
- 非显而易见的业务规则。
- 常见陷阱和暂时不能修改的历史设计。

## 不应该包含什么

通常不需要写入项目级 `CLAUDE.md` 的内容包括：

- 能从构建文件直接读取的依赖列表。
- 完整目录树。
- formatter 和 linter 已经强制执行的规则。
- README 中已有的项目介绍。
- 用户级 `CLAUDE.md` 已经定义的沟通方式和 Git 工作流。
- 本机用户名、绝对路径、密钥和其他个人环境信息。
- 面向模板使用者的初始化教程。

## 维护原则

- `CLAUDE.md` 保持为可直接复制的纯模板。
- 本文件负责说明模板的使用方法。
- 项目经验只有在具备普遍复用价值时，才回写到母版。
- 某个项目独有的规则只保留在该项目中。
- 修改模板结构后，同步检查本说明和仓库根目录 `README.md`。
